On Procs, and Lambdas

September 6, 2015

Procs and Lambdas are callable objects. A proc is an object that holds code sequences--it's an object that can be passed as a method argument, and you can also run the code when you use the call method. A lambda is a type of proc, but there are subtle differences. To be clear, a proc is not a method--it is still an object, but with code sequences stored. So what does that look like?

Here are the basics to creating and calling procs. Similar to when you create new arrays or class instances, you call the new method on a proc, while including the code sequence (similar to a block) you want the proc to hold, like so:

say_hi = Proc.new {puts "Hi!"} say_hi.call # => Hi!

When calling the proc, you treat it like an object, passing it like an argument. You can call it using the call method, as seen above. Or, in the example below, you see the & character; it is equivalent to the to_prov method, which treats the block-like proc into an object-like proc in order to pass it as an argument in a method, like so:

class Add attr_accessor :numbers def self.to_proc Proc.new {|i| i + 2} end end test = Add.new test.numbers = [1, 2, 3] p test.numbers.collect!(&Add) # => [3, 4, 5]

There are other aspects about procs that we can explore later, but I want to jump into lambdas as well. As I mentioned before, the two are quite similar. In fact, here's an example that looks almost exactly like a proc!

whaddup = lambda { puts "Whaddup"} whaddup.call # => Whaddup

You create and call a lambda in the same you would a proc. But, when you create a lambda, the irb identifies it as a proc. Still, at least from what I've read, lambda are different in three ways. First, lambdas are explicitly created; that is, if you call a block in a method, unless you specifically call a lambda, Ruby assumes it is just a regular proc. Second, when you call a return in a lambda, it will only exit out of the lambda but continues with the rest of the code in the method it was called in. In contrast, when you call a return in a proc, it will exit out the method entirely. Finally, lambda only answer when the exact number of arguments required are called. In short, procs and lambdas are almost identical, but lambdas are a bit more high maintenance and particular.
