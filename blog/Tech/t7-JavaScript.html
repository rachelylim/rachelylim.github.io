Ruby vs JS: On Loops

September 11, 2015

We started learning JavaScript (JS) this week. It's really ugly and petty. Somehow, this strange and hideous language is considered to be one of the "hottest" languages in the developer community, and if you master this language, you can probably get a fancy job. In this entry, I'll compare two simple while loop, one in Ruby, the other in JS, both of which will accomplish the same thing. After I compare/contrast the two, you decide--Team Ruby or Team JS?

The whileloops below are just going to print numbers 1 through 5.

#Ruby Loop i = 1 while i <= 5 print i i += 1 end # => 1, 2, 3, 4, 5 //JavaScript Loop var i = 1 while(i <= 5) { console.log(i); i++; }; //=> 1, 2, 3, 4, 5

Let's quickly run through the Ruby loop. We create a variable i to represent an integer, set it equal to one. We start the while loop. In the loop, we tell Ruby to print the integer, then increment the integer up by one, while the integer is less than or equal to 5. The result is the numbers 1 through 5 printed in the console. Pretty straightforward.

The JS loop also starts off by creating an integer. To do this, we need to use the var method to create a new integer, and set it equal to 1. We then create a while loop, and in parentheses, we define that the loop must continue while the integer is less than or equal to 5. Before moving on to what the method will do, we must open a curly bracket. Inside the curly bracket, which holds the action, we use the console.log() in order to print the integer on the console. After printing the integer, we also increment the integer by one using the nifty shortcut ++ after the integer. You may have noticed that each line is followed by a semicolon, which is required for "proper" syntax (although, I think the code runs without the semicolons sometimes). Finally, we end the loop with a closing curly bracket in the next line, which also ends with a semicolon.

Compare my paragraph walking through the Ruby loop to the paragraph describing the Javascript loop, and you can tell that Ruby is quite elegant and clean. But further experience in creating objects and methods in JS has made me realize that strengths in this barbaric language. There are some wonderful, flexible qualities about JS objects and function that make it ideal for creating games, which I hope to share with you in the near future.
